1 The EML Operator

Odrzywołek (2026) discovered that a single binary operator, paired with the constant $1$, can generate every standard elementary function — the continuous-math analog of the NAND gate for Boolean logic.

(1) $$\eml(x, y) \;=\; \exp(x) - \ln(y)$$

The grammar is context-free and trivially simple:

$$S \;\to\; 1 \;\mid\; \eml(S, S)$$

Every elementary expression becomes a binary tree of identical nodes. Key identities include:

$\ln(x)$$\eml\!\bigl(1,\;\eml(\eml(1, x),\; 1)\bigr)$3

$x \times y$depth-8 composition8

Internally, computations proceed over $\C$ via the principal branch of the complex logarithm. Trigonometric functions emerge through Euler's formula: $e^{i\varphi} = \cos\varphi + i\sin\varphi$, with $i = \sqrt{-1}$ obtainable from $\ln(-1) = i\pi$.

2 Koopman Operator Background

Consider a discrete-time nonlinear dynamical system on a state space $\mathcal{M} \subseteq \R^n$:

(2) $$\bx_{k+1} = F(\bx_k)$$

The Koopman operator $\mathcal{K}$ acts on scalar-valued observables $g : \mathcal{M} \to \C$ by composition with the dynamics:

(3) $$[\mathcal{K}\, g](\bx) \;=\; g\bigl(F(\bx)\bigr)$$

This is an infinite-dimensional linear operator, even when $F$ is nonlinear. The key insight: if we find a finite set of observables $\bg(\bx) = [g_1(\bx), \ldots, g_N(\bx)]^\top$ that span a $\mathcal{K}$-invariant subspace, then the dynamics in lifted coordinates become:

(4) $$\bg(\bx_{k+1}) \;=\; \bK \;\bg(\bx_k)$$

where $\bK \in \C^{N \times N}$ is a finite-dimensional Koopman matrix. The central problem is choosing the dictionary of observables $\{g_j\}$. Traditional approaches (EDMD, kernel EDMD) hand-select bases — polynomials, RBFs, Fourier modes, delay coordinates — and hope they span the relevant invariant subspace. This is ad hoc and offers no completeness guarantee.

3 EML Trees as Koopman Observables

Core idea: Use parameterized EML trees as a learnable, complete dictionary of observable functions. Since every elementary function is representable as an EML tree, the search space is guaranteed to contain the true Koopman eigenfunctions whenever they are elementary — which holds for a wide class of physically-motivated systems (harmonic oscillators, Lotka–Volterra, pendula, etc.).

3.1 The Master Formula

Following Section 4.3 of the paper, each input to an EML node is parameterized as:

(5) $$\text{input}_i \;=\; \alpha_i + \beta_i \cdot x + \gamma_i \cdot f_{\text{child}}$$

where $(\alpha_i, \beta_i, \gamma_i)$ are trainable weights that select among:

Selection$\alpha_i$$\beta_i$$\gamma_i$Result
Constant 1100$1$
Input variable010$x$
Child output001$f_{\text{child}}$

During training, $(\alpha, \beta, \gamma)$ are treated as Gumbel-softmax logits normalized to a simplex. Temperature $\tau$ anneals from soft (exploration) to hard (discrete). After convergence, the weights snap to one-hot $\{0, 1\}$, yielding an exact symbolic expression.

3.2 Full Level-2 Example

The complete level-2 master formula (14 parameters for univariate $x$) is:

(6) $$F(x) = \eml\!\Bigl[\;\alpha_1 + \beta_1 x + \gamma_1\,\eml(\alpha_3 + \beta_3 x,\;\alpha_4 + \beta_4 x),\;\;\alpha_2 + \beta_2 x + \gamma_2\,\eml(\alpha_5 + \beta_5 x,\;\alpha_6 + \beta_6 x)\;\Bigr]$$

Setting $\alpha_1\!=\!0, \beta_1\!=\!1, \gamma_1\!=\!0$ and $\alpha_2\!=\!1, \beta_2\!=\!\gamma_2\!=\!0$ recovers $\exp(x)$. The general depth-$d$ formula has $5 \cdot 2^d - 6$ parameters.

3.3 Koopman Dictionary Construction

Construct $N$ independent EML trees, each of depth $d$, to form the observable dictionary:

(7) $$\bg_\theta(\bx) \;=\; \bigl[\,T_1^\theta(\bx),\;\; T_2^\theta(\bx),\;\; \ldots,\;\; T_N^\theta(\bx)\,\bigr]^\top$$

where $T_j^\theta$ denotes the $j$-th EML tree with trainable routing parameters $\theta$. The full Koopman model is then:

(8) $$\bg_\theta(\bx_{k+1}) \;\approx\; \bK\;\bg_\theta(\bx_k), \qquad \bx_k \;\approx\; \bC\;\bg_\theta(\bx_k)$$

where $\bK \in \R^{N \times N}$ is the Koopman matrix and $\bC \in \R^{n \times N}$ is the decoder (reconstruction map).

4 Taylor-Series GPU Primitives

The key performance insight: both $\exp$ and $\ln$ can be replaced by polynomial approximations in Horner form, reducing the entire EML operator to fused multiply-add (FMA) instructions with zero transcendental function calls. This is critical for GPU throughput, especially with nested tree compositions where each node calls EML.

4.1 Taylor Exponential with Range Reduction

Decompose $x = k \ln 2 + r$ where $k = \lfloor x / \ln 2 \rceil$ and $|r| \le \frac{\ln 2}{2} \approx 0.347$. Then:

(9) $$\exp(x) = 2^k \cdot \exp(r), \qquad \exp(r) = \sum_{n=0}^{M} \frac{r^n}{n!}$$

The Taylor series is evaluated in Horner form (innermost term outward):

(10) $$\exp(r) \;=\; 1 + r\!\left(1 + \frac{r}{2}\!\left(1 + \frac{r}{3}\!\left(1 + \cdots \frac{r}{M}\right)\!\right)\!\right)$$

Each step is one multiply-add. With $|r| \le 0.347$, order $M = 12$ gives error $\lesssim 10^{-16}$ (double precision). The $2^k$ scaling is a single ldexp-equivalent operation.

4.2 Taylor Logarithm via Arctanh

Range-reduce: $y = m \cdot 2^e$ with $m \in [0.5, 1.0)$, so $\ln(y) = e \ln 2 + \ln(m)$. Then use the arctanh series, which converges much faster than the naive $\ln(1+t)$ expansion:

(11) $$\ln(m) \;=\; 2\,\operatorname{arctanh}\!\left(\frac{m-1}{m+1}\right) \;=\; 2\sum_{k=0}^{M}\frac{u^{2k+1}}{2k+1}, \qquad u = \frac{m-1}{m+1}$$

For $m \in [0.5, 1)$ we have $|u| \le \tfrac{1}{3}$, so this converges rapidly. Again evaluated via Horner:

(12) $$\sum_{k=0}^{M} \frac{u^{2k+1}}{2k+1} \;=\; u\!\left(\frac{1}{1} + u^2\!\left(\frac{1}{3} + u^2\!\left(\frac{1}{5} + \cdots\right)\!\right)\!\right)$$

Order $M = 16$ gives machine-epsilon accuracy for $\ln$.

4.3 Combined EML Primitive

The GPU-optimized EML is then:

(13) $$\eml_{\text{Taylor}}(x, y) \;=\; \text{horner\_exp}(x, M_e) \;-\; \text{horner\_ln}(y, M_l)$$

with gradients $\frac{\partial}{\partial x}\eml = \exp(x)$ and $\frac{\partial}{\partial y}\eml = -\frac{1}{y}$, both computable via the same polynomial machinery. The custom torch.autograd.Function avoids redundant recomputation in the backward pass.

Performance note: For a depth-$d$ tree with $N$ parallel observables evaluated over a batch of size $B$, the forward pass requires $d$ sequential EML calls, each operating on tensors of shape $[B, N, 2^{d-1-l}]$ at level $l$. All nodes at the same depth are computed in a single batched call — no Python loops over individual nodes.

5 Vectorized Tree Architecture

5.1 Routing via Gumbel-Softmax

At each EML node $i$, both the left and right inputs are selected from a candidate set $\mathcal{C}_i = \{1, x_0, x_1, \ldots, x_{n-1}\}$ (for leaves) or $\mathcal{C}_i = \{1, x_0, \ldots, x_{n-1}, f_{\text{child}}\}$ (for interior nodes). The selection is parameterized by logits $\ell_i \in \R^{|\mathcal{C}_i|}$:

(14) $$w_i^{(j)} = \frac{\exp\!\bigl((\ell_i^{(j)} + G_j)/\tau\bigr)}{\sum_k \exp\!\bigl((\ell_i^{(k)} + G_k)/\tau\bigr)}, \qquad G_j \sim \text{Gumbel}(0,1)$$

The selected input is $\sum_j w_i^{(j)} c_j$ where $c_j \in \mathcal{C}_i$. As $\tau \to 0$, this converges to hard one-hot selection.

5.2 Tensor Layout

All $N$ trees are packed into contiguous parameter tensors, one per tree level:

$$\text{level\_logits}[l] \;\in\; \R^{N \times n_l \times |\mathcal{C}_l| \times 2}$$

where $n_l = 2^{d-1-l}$ is the number of nodes at level $l$, and the trailing dimension 2 indexes left/right inputs. Bottom-up evaluation proceeds as $d$ sequential batched operations:

Algorithm 1: Vectorized EML Tree Evaluation

  1. for $l = 0, 1, \ldots, d-1$ (bottom to top):
  2.   Compute Gumbel-softmax weights: $W \in \R^{B \times N \times n_l \times |\mathcal{C}_l| \times 2}$
  3.   Assemble candidate tensors (const $1$, input vars, child outputs from level $l\!-\!1$)
  4.   Weighted selection: $\text{eml\_x}, \text{eml\_y} \in \R^{B \times N \times n_l}$
  5.   Clamp inputs: $\text{eml\_x} \gets \text{clamp}(\text{eml\_x}, -C, C)$
  6.   Batched EML: $\text{out}_l = \eml_{\text{Taylor}}(\text{eml\_x},\; \text{eml\_y})$
  7. return $\text{out}_{d-1} \in \R^{B \times N}$  (root outputs = lifted state)

6 Training Pipeline

6.1 Loss Function

Given trajectory data $\{(\bx_k, \bx_{k+1})\}_{k=1}^{K}$, the joint objective is:

(15) $$\mathcal{L}(\theta, \bK, \bC) \;=\; \underbrace{\sum_k \bigl\|\bg_\theta(\bx_{k+1}) - \bK\,\bg_\theta(\bx_k)\bigr\|^2}_{\text{prediction}} \;+\; \lambda\,\underbrace{\sum_k \bigl\|\bx_k - \bC\,\bg_\theta(\bx_k)\bigr\|^2}_{\text{reconstruction}} \;+\; \mu\,\underbrace{\|\bK\|_F^2}_{\text{regularization}}$$

All three components are differentiable through the EML trees (the Taylor polynomials have clean analytic gradients). Standard backpropagation with Adam applies.

6.2 Three-Phase Annealing Schedule

Algorithm 2: Training Protocol

  1. Phase 1 — Exploration (epochs $0$ to $0.6T$):
    Anneal $\tau$ from $\tau_{\max}$ (e.g., 2.0) to $1.0$. All parameters ($\theta, \bK, \bC$) are trainable. Gumbel noise encourages exploration of different routing configurations.
  2. Phase 2 — Hardening (epochs $0.6T$ to $0.9T$):
    Anneal $\tau$ from $1.0$ to $\tau_{\min}$ (e.g., 0.1). Routing weights sharpen toward discrete selections. The trees commit to specific functional forms.
  3. Phase 3 — Snap & Fine-Tune (epochs $0.9T$ to $T$):
    Snap all routing logits to hard one-hot (argmax). Freeze tree parameters $\theta$. Fine-tune only $\bK$ and $\bC$ at reduced learning rate.
Overflow hazard: Nested EML compositions produce iterated exponentials ($e^{e^{e^x}}$) during exploration. Clamp EML inputs to $[-C, C]$ with $C \approx 20$ and normalize state data to $[-1, 1]$ before lifting.

6.3 Warm-Start Strategies

The paper reports that convergence from random initialization degrades sharply beyond depth 4. For Koopman applications, two warm-start strategies can dramatically improve recovery:

  1. EDMD bootstrap: Run standard EDMD with a polynomial dictionary first. Initialize EML tree routing to mimic the best polynomial observables (since polynomials are elementary, they have EML representations).
  2. Curriculum depth: Train at depth 2 until convergence, then grow each tree by one level (appending new leaf nodes with random logits) and continue training. The existing structure provides a strong initialization for the expanded tree.

7 Eigendecomposition & Interpretability

After training and snapping, we have an explicit Koopman matrix $\bK$. Its eigendecomposition:

(16) $$\bK = \bV \,\Lambda\, \bV^{-1}, \qquad \Lambda = \operatorname{diag}(\lambda_1, \ldots, \lambda_N)$$

yields:

  • Koopman eigenvalues $\lambda_j$: encode frequencies ($\omega_j = \operatorname{Im}(\ln \lambda_j)$) and growth/decay rates ($\sigma_j = \operatorname{Re}(\ln \lambda_j)$).
  • Koopman eigenfunctions: $\varphi_j(\bx) = \mathbf{v}_j^\top\, \bg_\theta(\bx)$, i.e., linear combinations of the snapped EML observables. Since each $g_i$ is now a closed-form elementary expression, $\varphi_j$ is also elementary — fully interpretable.
  • Koopman modes: $\mathbf{m}_j = \bC\,\mathbf{v}_j$, linking each eigenfunction to its contribution in state space.

This is a fundamental advantage over deep-learning Koopman approaches (e.g., LRAN, DeepKoopman): instead of opaque neural network observables, we get symbolic formulas that can be analyzed, simplified, and compared to physics.

8 Connection to Information Geometry

The Koopman eigenfunctions $\{\varphi_j\}$ define a natural coordinate system on the state manifold $\mathcal{M}$. The lifting map $\bg_\theta : \mathcal{M} \to \R^N$ induces a pullback metric on $\mathcal{M}$:

(17) $$(G_\theta)_{ab} \;=\; \sum_{j=1}^{N} \frac{\partial g_j}{\partial x^a}\,\frac{\partial g_j}{\partial x^b}$$

When $\bg_\theta$ consists of snapped EML trees, each $g_j$ is a known elementary function, so $G_\theta$ is computable in closed form. This connects directly to the Fisher information metric if the observables are interpreted as sufficient statistics of a parametric family.

For multi-agent RL systems (e.g., Bug World), this enables:

  • Behavioral basis decomposition: The Koopman eigenfunctions separate the dynamics into independent modes. Each mode corresponds to a distinct "behavioral basis" — e.g., pursuit, evasion, foraging — with its own characteristic timescale $1/\sigma_j$ and frequency $\omega_j$.
  • Natural gradient on policy manifolds: The pullback metric $G_\theta$ provides the Riemannian structure needed for natural gradient descent on the policy manifold. With EML observables, this metric has an explicit elementary form rather than requiring numerical estimation of the Fisher matrix.
  • Geodesic policy interpolation: Geodesics on $(\mathcal{M}, G_\theta)$ define "shortest paths" between behavioral strategies, enabling principled policy interpolation and transfer.

9 Open Questions & Next Steps

9.1 Immediate Implementation Tasks

  1. Complex mode: Predator-prey dynamics (Bug World) have oscillatory Koopman eigenfunctions. The implementation needs complex128 Taylor primitives throughout the tree, with final projection to reals. The complex exp/ln Taylor series are straightforward extensions but need careful handling of branch cuts.
  2. Benchmarking: Compare EML dictionary against polynomial EDMD and deep Koopman (autoencoder) on standard systems: Duffing oscillator, Van der Pol, Lorenz, and Lotka-Volterra. Metrics: prediction horizon, reconstruction error, eigenfunction interpretability.
  3. Scaling: Profile GPU utilization for $N = 64$ trees at depth 5 ($\sim$10K params). The bottleneck is likely the $d$ sequential levels, not the within-level parallelism.

9.2 Research Questions

  1. Can EML trees discover non-obvious Koopman eigenfunctions that polynomial/RBF dictionaries miss? (e.g., $g(x) = \ln(1 + e^{x^2})$ type observables)
  2. Does the EML master formula provide a better loss landscape for Koopman learning than neural network encoders? (The discrete-snap property suggests yes for shallow trees.)
  3. Can the ternary EML variant $T(x,y,z) = e^x / \ln x \times \ln z / e^y$ (requiring no distinguished constant) improve training convergence?
  4. Is there a connection between EML tree depth and the spectral complexity of the Koopman operator for a given system?

9.3 Longer-Term Directions

  1. Continuous-depth EML trees (neural ODE analog): parameterize tree growth as a continuous process, enabling adaptive-depth observables.
  2. EML-KAN hybrid: Use EML nodes as the activation functions in a Kolmogorov–Arnold network, combining KAN's universal approximation with EML's symbolic snap property.
  3. Hardware EML: The paper notes potential for FPGA/analog circuit implementation. A dedicated EML coprocessor could evaluate deep trees in $O(d)$ clock cycles.
