\title{Koopman Operator Lifting via EML Observable Dictionaries:\\

\author{Jim Beaver}
\date{April 2026 --- Working Draft}

\begin{abstract}
We introduce a framework for learning interpretable Koopman operator
representations using the EML (Exp-Minus-Log) Sheffer operator
$\eml(x,y) = \exp(x) - \ln(y)$ as the basis for observable dictionaries.
Since every elementary function can be expressed as a binary tree of identical
EML nodes, parameterized EML trees provide a \emph{complete}, gradient-trainable
search space for Koopman eigenfunctions.  After Gumbel-softmax annealing and
weight snapping, the learned observables are exact symbolic expressions ---
yielding closed-form Koopman eigenfunctions that black-box methods cannot produce.
We replace transcendental function calls with Horner-form Taylor-series primitives,
reducing the entire EML operator to fused multiply-add instructions for maximum GPU
throughput.  The resulting model is packaged as a PySINDy-compatible feature library
and validated against the dynamicsai.org Common Task Framework (CTF) 12-metric
benchmark on the Lorenz system, comparing to EDMD, deep Koopman autoencoders, and
PySR baselines.
\end{abstract}

\section{Introduction}
\label{sec:intro}

The Koopman operator provides a linear (but infinite-dimensional) description of
nonlinear dynamical systems \citep{koopman1931,mezic2005,budisic2012}.  A central challenge is selecting
the dictionary of observable functions that span a finite-dimensional invariant
subspace.  Traditional approaches --- polynomials, radial basis functions, Fourier
modes, delay coordinates --- are chosen ad hoc and offer no completeness guarantee.
Deep-learning methods \citep{takeishi2017,lusch2018} learn flexible representations
but sacrifice interpretability: the resulting observables are opaque neural
networks with no closed-form expressions.  Symbolic regression
\citep{schmidt2009,cranmer2023} can discover closed-form expressions, but existing
approaches lack the completeness guarantee that EML provides.

\citet{odrzywołek2026} discovered that the binary operator
$\eml(x,y) = \exp(x) - \ln(y)$, paired with the constant~$1$, generates
\emph{every} standard elementary function --- the continuous-math analog of the NAND
gate \citep{sheffer1913} for Boolean logic.  This provides a natural, complete basis for Koopman
observable dictionaries.  Because every elementary function admits an EML tree
representation, the search space is guaranteed to contain the true Koopman
eigenfunctions whenever they are elementary --- which holds for a wide class of
physically-motivated systems (harmonic oscillators, Lotka--Volterra, pendula, etc.).

After training with Gumbel-softmax annealing \citep{jang2017}, the continuous routing parameters snap
to discrete one-hot selections, yielding \emph{exact symbolic expressions}.  The
resulting Koopman eigenfunctions are linear combinations of these snapped EML
observables and are therefore also elementary functions --- fully interpretable and
amenable to analytic manipulation.

\paragraph{Contributions.}
\begin{itemize}
    \item Parameterized EML trees as gradient-trainable Koopman observables with
          exact symbolic snap, providing a \emph{complete} dictionary over elementary
          functions.
    \item GPU-optimized Taylor-series EML primitives in Horner form with range
          reduction, achieving zero transcendental function calls in both forward
          and backward passes.
    \item A fully vectorized tree architecture that evaluates all dictionary elements
          in a single batched call per tree level.
    \item A three-phase training pipeline (exploration, hardening, snap \& fine-tune)
          with Gumbel-softmax annealing.
    \item PySINDy integration via \texttt{EMLLibrary}.
    \item CTF benchmark evaluation (E1--E12) on Lorenz, with comparison to EDMD,
          deep Koopman, and PySR baselines.
\end{itemize}

\section{Background}
\label{sec:background}

\subsection{The EML Operator}
\label{sec:eml}

\citet{odrzywołek2026} introduced a binary operator that, together with the
constant~$1$, generates every standard elementary function.  The operator is
defined as:
\begin{equation}\label{eq:eml}
  \eml(x, y) \;=\; \exp(x) - \ln(y).
\end{equation}
The grammar is context-free and trivially simple:
\begin{equation}\label{eq:grammar}
  S \;\to\; 1 \;\mid\; \eml(S, S).
\end{equation}
Every elementary expression therefore becomes a \emph{binary tree of identical
nodes}.  Key identities include:

\begin{center}
\begin{tabular}{lcl}
\toprule
Function & EML Form & Depth \\
\midrule
$e^x$ & $\eml(x, 1)$ & 1 \\
$e$ & $\eml(1, 1)$ & 1 \\
$\ln(x)$ & $\eml\!\bigl(1,\;\eml(\eml(1, x),\; 1)\bigr)$ & 3 \\
$-x$ & depth-5 composition & 5 \\
$x \times y$ & depth-8 composition & 8 \\
\bottomrule
\end{tabular}
\end{center}

Internally, computations proceed over~$\C$ via the principal branch of the complex
logarithm.  Trigonometric functions emerge through Euler's formula:
$e^{i\varphi} = \cos\varphi + i\sin\varphi$, with $i = \sqrt{-1}$ obtainable from
$\ln(-1) = i\pi$.


\subsection{Koopman Operator Theory}
\label{sec:koopman}

Consider a discrete-time nonlinear dynamical system on a state space
$\cM \subseteq \R^n$:
\begin{equation}\label{eq:dynamics}
  \bx_{k+1} = F(\bx_k).
\end{equation}
The \textbf{Koopman operator}~$\cK$ acts on scalar-valued observables
$g : \cM \to \C$ by composition with the dynamics:
\begin{equation}\label{eq:koopman}
  [\cK\, g](\bx) \;=\; g\bigl(F(\bx)\bigr).
\end{equation}
This is an infinite-dimensional linear operator, even when~$F$ is nonlinear.  The
key insight: if we find a finite set of observables
$\bg(\bx) = [g_1(\bx), \ldots, g_N(\bx)]^\top$ that span a $\cK$-invariant
subspace, then the dynamics in lifted coordinates become:
\begin{equation}\label{eq:lifted}
  \bg(\bx_{k+1}) \;=\; \bK \;\bg(\bx_k),
\end{equation}
where $\bK \in \C^{N \times N}$ is a finite-dimensional \textbf{Koopman matrix}.
The central problem is choosing the dictionary of observables~$\{g_j\}$.

\textbf{Extended Dynamic Mode Decomposition (EDMD)} \citep{williams2015} solves
for~$\bK$ by least-squares regression given a fixed, hand-selected dictionary.
The original DMD algorithm \citep{schmid2010,tu2014} uses linear observables;
EDMD extends this to nonlinear dictionaries.  Typical choices --- polynomials,
radial basis functions, Fourier modes --- are ad hoc and offer no guarantee that
the relevant invariant subspace is spanned.  Kernel EDMD
\citep{williams2015kernel} and deep Koopman autoencoders \citep{lusch2018,otto2019}
address expressiveness but sacrifice either tractability or interpretability.
For a comprehensive review see \citet{brunton2022modern}.


\subsection{SINDy and PySINDy}
\label{sec:sindy}

The Sparse Identification of Nonlinear Dynamics (SINDy)
framework \citep{brunton2016} discovers governing equations by regressing
time derivatives against a library of candidate nonlinear functions using
sparsity-promoting optimization.  PySINDy \citep{desilva2020,kaptanoglu2022}
provides a modular Python implementation with interchangeable feature libraries.

Our approach connects to SINDy through the shared notion of a feature library:
rather than using pre-defined polynomial or Fourier features, we supply EML trees
as the library.  After symbolic snap, the discovered features can be fed into a
standard SINDy regression pipeline, combining EML's completeness guarantee with
SINDy's sparse model selection.

\section{Method}
\label{sec:method}

\subsection{EML Trees as Koopman Observables}
\label{sec:eml-trees}

\subsubsection{The Master Formula}

Each input to an EML node is parameterized as a soft selection among a candidate
set:
\begin{equation}\label{eq:master}
  \text{input}_i \;=\; \alpha_i + \beta_i \cdot x + \gamma_i \cdot f_{\text{child}},
\end{equation}
where $(\alpha_i, \beta_i, \gamma_i)$ are trainable weights that select among:

\begin{center}
\begin{tabular}{lcccc}
\toprule
Selection & $\alpha_i$ & $\beta_i$ & $\gamma_i$ & Result \\
\midrule
Constant $1$ & 1 & 0 & 0 & $1$ \\
Input variable & 0 & 1 & 0 & $x$ \\
Child output & 0 & 0 & 1 & $f_{\text{child}}$ \\
\bottomrule
\end{tabular}
\end{center}

During training, $(\alpha, \beta, \gamma)$ are treated as \textbf{Gumbel-softmax
logits} \citep{jang2017} normalized to a simplex.  Temperature~$\tau$ anneals from soft
(exploration) to hard (discrete).  After convergence, the weights snap to one-hot
$\{0, 1\}$, yielding an exact symbolic expression.

More generally, at each EML node~$i$, both left and right inputs are selected from
a candidate set
$\cC_i = \{1, x_0, x_1, \ldots, x_{n-1}\}$ (for leaves) or
$\cC_i = \{1, x_0, \ldots, x_{n-1}, f_{\text{child}}\}$ (for interior nodes).
The selection is parameterized by logits $\ell_i \in \R^{|\cC_i|}$:
\begin{equation}\label{eq:gumbel}
  w_i^{(j)} = \frac{\exp\!\bigl((\ell_i^{(j)} + G_j)/\tau\bigr)}
  {\sum_k \exp\!\bigl((\ell_i^{(k)} + G_k)/\tau\bigr)},
  \qquad G_j \sim \text{Gumbel}(0,1).
\end{equation}
The selected input is $\sum_j w_i^{(j)} c_j$ where $c_j \in \cC_i$.  As
$\tau \to 0$, this converges to hard one-hot selection.

\subsubsection{Full Level-2 Example}

The complete level-2 master formula (14 parameters for univariate~$x$) is:
\begin{equation}\label{eq:level2}
\begin{split}
  F(x) = \eml\Bigl[\;
    &\alpha_1 + \beta_1 x + \gamma_1\,\eml(\alpha_3 + \beta_3 x,\;\alpha_4 + \beta_4 x), \\
    &\alpha_2 + \beta_2 x + \gamma_2\,\eml(\alpha_5 + \beta_5 x,\;\alpha_6 + \beta_6 x)
  \;\Bigr].
\end{split}
\end{equation}
Setting $\alpha_1\!=\!0, \beta_1\!=\!1, \gamma_1\!=\!0$ and
$\alpha_2\!=\!1, \beta_2\!=\!\gamma_2\!=\!0$ recovers $\exp(x)$.  The general
depth-$d$ formula has $5 \cdot 2^d - 6$ parameters.

\subsubsection{Koopman Dictionary Construction}

We construct $N$ independent EML trees, each of depth~$d$, to form the observable
dictionary:
\begin{equation}\label{eq:dictionary}
  \bg_\theta(\bx) \;=\; \bigl[\,T_1^\theta(\bx),\;\; T_2^\theta(\bx),\;\;
  \ldots,\;\; T_N^\theta(\bx)\,\bigr]^\top,
\end{equation}
where $T_j^\theta$ denotes the $j$-th EML tree with trainable routing
parameters~$\theta$.  The full Koopman model is then:
\begin{equation}\label{eq:koopman-model}
  \bg_\theta(\bx_{k+1}) \;\approx\; \bK\;\bg_\theta(\bx_k), \qquad
  \bx_k \;\approx\; \bC\;\bg_\theta(\bx_k),
\end{equation}
where $\bK \in \R^{N \times N}$ is the Koopman matrix and
$\bC \in \R^{n \times N}$ is the decoder (reconstruction map).

\subsubsection{Vectorized Tree Architecture}

All $N$ trees are packed into contiguous parameter tensors, one per tree level:
\begin{equation}
  \text{level\_logits}[l] \;\in\; \R^{N \times n_l \times |\cC_l| \times 2},
\end{equation}
where $n_l = 2^{d-1-l}$ is the number of nodes at level~$l$ and the trailing
dimension~2 indexes left/right inputs.  Bottom-up evaluation proceeds as $d$
sequential batched operations (Algorithm~\ref{alg:tree-eval}).

\begin{algorithm}[t]
\caption{Vectorized EML Tree Evaluation}
\label{alg:tree-eval}
\begin{algorithmic}[1]
\REQUIRE State batch $\bx \in \R^{B \times n}$, tree depth $d$, temperature $\tau$
\ENSURE Lifted observables $\bg \in \R^{B \times N}$
\FOR{$l = 0, 1, \ldots, d-1$ (bottom to top)}
  \STATE Compute Gumbel-softmax weights
    $W \in \R^{B \times N \times n_l \times |\cC_l| \times 2}$
  \STATE Assemble candidate tensors (const $1$, input vars, child outputs from
    level $l\!-\!1$)
  \STATE Weighted selection:
    $\text{eml\_x}, \text{eml\_y} \in \R^{B \times N \times n_l}$
  \STATE Clamp inputs:
    $\text{eml\_x} \gets \text{clamp}(\text{eml\_x}, -C, C)$
  \STATE Batched EML:
    $\text{out}_l = \eml_{\text{Taylor}}(\text{eml\_x},\; \text{eml\_y})$
\ENDFOR
\RETURN $\text{out}_{d-1} \in \R^{B \times N}$ (root outputs = lifted state)
\end{algorithmic}
\end{algorithm}

\subsection{Taylor-Series GPU Primitives}
\label{sec:taylor}

Both $\exp$ and $\ln$ can be replaced by polynomial approximations in Horner form,
reducing the entire EML operator to fused multiply-add (FMA) instructions with zero
transcendental function calls.  This is critical for GPU throughput, especially with
nested tree compositions where each node calls EML.

\subsubsection{Taylor Exponential with Range Reduction}

Decompose $x = k \ln 2 + r$ where $k = \lfloor x / \ln 2 \rceil$ and
$|r| \le \frac{\ln 2}{2} \approx 0.347$.  Then:
\begin{equation}\label{eq:exp-range}
  \exp(x) = 2^k \cdot \exp(r), \qquad
  \exp(r) = \sum_{n=0}^{M} \frac{r^n}{n!}.
\end{equation}
The Taylor series is evaluated in Horner form (innermost term outward):
\begin{equation}\label{eq:exp-horner}
  \exp(r) \;=\; 1 + r\!\left(1 + \frac{r}{2}\!\left(1 + \frac{r}{3}\!\left(1
  + \cdots \frac{r}{M}\right)\!\right)\!\right).
\end{equation}
Each step is one multiply-add.  With $|r| \le 0.347$, order $M = 12$ gives error
$\lesssim 10^{-16}$ (double precision).  The $2^k$ scaling is a single
\texttt{ldexp}-equivalent operation.

\subsubsection{Taylor Logarithm via Arctanh}

Range-reduce: $y = m \cdot 2^e$ with $m \in [0.5, 1.0)$, so
$\ln(y) = e \ln 2 + \ln(m)$.  Then use the arctanh series, which converges much
faster than the naive $\ln(1+t)$ expansion:
\begin{equation}\label{eq:ln-arctanh}
  \ln(m) \;=\; 2\,\operatorname{arctanh}\!\left(\frac{m-1}{m+1}\right)
  \;=\; 2\sum_{k=0}^{M}\frac{u^{2k+1}}{2k+1}, \quad u = \frac{m-1}{m+1}.
\end{equation}
For $m \in [0.5, 1)$ we have $|u| \le \tfrac{1}{3}$, giving rapid convergence.
Horner evaluation:
\begin{equation}\label{eq:ln-horner}
  \sum_{k=0}^{M} \frac{u^{2k+1}}{2k+1} \;=\; u\!\left(\frac{1}{1} +
  u^2\!\left(\frac{1}{3} + u^2\!\left(\frac{1}{5} + \cdots\right)\!\right)\!\right).
\end{equation}
Order $M = 16$ gives machine-epsilon accuracy for $\ln$.

\subsubsection{Combined EML Primitive}

The GPU-optimized EML primitive is:
\begin{equation}\label{eq:eml-taylor}
  \eml_{\text{Taylor}}(x, y) \;=\; \text{horner\_exp}(x, M_e) \;-\;
  \text{horner\_ln}(y, M_l),
\end{equation}
with gradients $\frac{\partial}{\partial x}\eml = \exp(x)$ and
$\frac{\partial}{\partial y}\eml = -\frac{1}{y}$, both computable via the same
polynomial machinery.  A custom \texttt{torch.autograd.Function} \citep{paszke2019} avoids redundant
recomputation in the backward pass: the forward stores $x$ and $y$ and reuses the
Horner evaluation for gradient computation.

For a depth-$d$ tree with $N$ parallel observables evaluated over a batch of
size~$B$, the forward pass requires $d$ sequential EML calls, each operating on
tensors of shape $[B, N, 2^{d-1-l}]$ at level~$l$.  All nodes at the same depth
are computed in a single batched call --- no Python loops over individual nodes.


\subsection{PySINDy Integration}
\label{sec:pysindy}

We package the EML dictionary as a PySINDy-compatible feature library,
\texttt{EMLLibrary}, supporting two modes:

\begin{itemize}
  \item \textbf{Enumerative mode:} Generate all distinct EML trees up to a specified
    depth (filtered by algebraic equivalence).  This provides a finite but complete
    basis for SINDy regression at a chosen complexity level.
  \item \textbf{Pre-trained mode:} Supply trees already trained via
    Gumbel-softmax Koopman optimization (Section~\ref{sec:training}).
    After snapping, each tree is a fixed symbolic feature.  SINDy's
    sparsity-promoting regression then selects the most relevant subset.
\end{itemize}

\noindent
Both modes implement the standard PySINDy \texttt{BaseFeatureLibrary} interface,
allowing seamless composition with existing PySINDy optimizers (STLSQ, SR3, etc.).

\section{Experimental Setup}
\label{sec:experiments}

\subsection{Training Pipeline}
\label{sec:training}

\subsubsection{Loss Function}

Given trajectory data $\{(\bx_k, \bx_{k+1})\}_{k=1}^{K}$, the joint objective is:
\begin{equation}\label{eq:loss}
  \cL(\theta, \bK, \bC) \;=\;
  \underbrace{\sum_k \bigl\|\bg_\theta(\bx_{k+1}) - \bK\,\bg_\theta(\bx_k)\bigr\|^2}%
  _{\text{prediction}}
  \;+\; \lambda\,
  \underbrace{\sum_k \bigl\|\bx_k - \bC\,\bg_\theta(\bx_k)\bigr\|^2}%
  _{\text{reconstruction}}
  \;+\; \mu\,
  \underbrace{\|\bK\|_F^2}_{\text{regularization}}.
\end{equation}
All three components are differentiable through the EML trees (the Taylor
polynomials have clean analytic gradients).

\subsubsection{Three-Phase Annealing Schedule}

Training proceeds in three phases:

\begin{enumerate}
  \item \textbf{Phase 1 --- Exploration} (epochs $0$ to $0.6T$):
    Anneal $\tau$ from $\tau_{\max}$ (e.g., 2.0) to~$1.0$.  All parameters
    $(\theta, \bK, \bC)$ are trainable.  Gumbel noise encourages exploration of
    different routing configurations.
  \item \textbf{Phase 2 --- Hardening} (epochs $0.6T$ to $0.9T$):
    Anneal $\tau$ from $1.0$ to $\tau_{\min}$ (e.g., 0.1).  Routing weights sharpen
    toward discrete selections; the trees commit to specific functional forms.
  \item \textbf{Phase 3 --- Snap \& Fine-Tune} (epochs $0.9T$ to $T$):
    Snap all routing logits to hard one-hot (argmax).  Freeze tree
    parameters~$\theta$.  Fine-tune only $\bK$ and $\bC$ at reduced learning rate.
\end{enumerate}

\noindent
\textbf{Overflow hazard.}  Nested EML compositions produce iterated exponentials
($e^{e^{e^x}}$) during exploration.  EML inputs are clamped to $[-C, C]$ with
$C = 5$ (so $\exp(C) \approx 148$), and state data is normalized to zero-mean
unit-variance before lifting.  The tight clamp is essential: $C = 20$ yields
$\exp(20) \approx 5 \times 10^8$, causing catastrophic loss magnitudes in early
training.

\subsubsection{Warm-Start Strategies}

Convergence from random initialization degrades sharply beyond depth~4.  Two
warm-start strategies improve recovery:

\begin{enumerate}
  \item \textbf{EDMD bootstrap:} Run standard EDMD with a polynomial dictionary
    first.  Initialize EML tree routing to mimic the best polynomial observables
    (since polynomials are elementary, they have EML representations).
  \item \textbf{Curriculum depth:} Train at depth~2 until convergence, then grow
    each tree by one level (appending new leaf nodes with random logits) and
    continue training.  The existing structure provides a strong initialization for
    the expanded tree.
\end{enumerate}


\subsection{CTF Benchmark Framework}
\label{sec:ctf}

We evaluate on the dynamicsai.org Common Task Framework
(CTF) \citep{wyder2025}, which defines
12 standardized metrics (E1--E12) for dynamical-system learning.  The benchmark
provides controlled comparisons across:

\begin{itemize}
  \item \textbf{E1 -- Short-term forecasting:} Normalized RMSE over 1--5
    Lyapunov-time horizons.
  \item \textbf{E2 -- Long-term statistics:} Error in time-averaged observables
    (mean, variance, autocorrelation).
  \item \textbf{E3 -- Attractor geometry:} Hausdorff distance between true and
    predicted attractors.
  \item \textbf{E4--E12:} Additional metrics covering Lyapunov exponents,
    power spectra, ergodic measures, and computational cost.
\end{itemize}

\noindent
We use the Lorenz-63 system ($\sigma\!=\!10$, $\rho\!=\!28$, $\beta\!=\!8/3$) as
the primary test case, with 10,000 training pairs sampled at $\Delta t = 0.01$.


\subsection{Baselines}
\label{sec:baselines}

We compare four methods:

\begin{enumerate}
  \item \textbf{EML-Koopman (ours):} $N\!=\!16$ trees at depth~2 (720
    parameters), three-phase training with $T\!=\!1200$ epochs, batch
    size~2048, input clamping at $\pm 5$.  Taylor orders: $M_e\!=\!10$,
    $M_l\!=\!12$.
  \item \textbf{EDMD-Poly:} Polynomial dictionary up to degree~4
    ($\binom{n+4}{4} = 35$ features for $n\!=\!3$), least-squares $\bK$.
  \item \textbf{EDMD-RBF:} 200 Gaussian RBF centers selected randomly from the
    training data, $\sigma\!=\!1.0$.
  \item \textbf{Deep Koopman:} 3-layer MLP autoencoder ($3 \to 128 \to 128
    \to 32$), symmetric decoder, linear dynamics in 32-dimensional latent
    space (43{,}299 parameters), trained for 1200 epochs.
\end{enumerate}

\paragraph{Long-term statistics (E2).}
EDMD-Poly achieves the highest E2 ($52.94$) by a wide margin, because its
polynomial basis functions naturally capture the power spectral shape of the
Lorenz attractor even when short-term prediction diverges.  EML-Koopman's E2
of $-7.54$ indicates room for improvement in long-term spectral fidelity, likely
through deeper trees (depth 3--4) that can express a wider range of nonlinear
functions.

\paragraph{Failure modes.}
EDMD-RBF diverges catastrophically (E1 $= -10{,}216$) because random Gaussian
centers in the normalized 3D Lorenz state space produce a severely ill-conditioned
Koopman matrix.  This underscores the importance of dictionary design --- a problem
that EML addresses through gradient-based observable discovery.

\subsection{Discovered Symbolic Observables}

After Phase~3 snap, all 16 EML tree observables reduce to closed-form
elementary functions.  Representative examples (with $x_0, x_1, x_2$
corresponding to normalized Lorenz $x, y, z$):

\begin{center}
\begin{tabular}{cl}
\toprule
Observable & Snapped Formula \\
\midrule
$g_0$ & $\eml(\eml(1, x_1),\; x_1)$ \\
$g_1$ & $\eml(\eml(1, x_1),\; 1)$ \\
$g_2$ & $\eml(\eml(1, x_0),\; 1)$ \\
$g_4$ & $\eml(\eml(1, x_1),\; x_2)$ \\
$g_{10}$ & $\eml(\eml(1, x_0),\; x_1)$ \\
\bottomrule
\end{tabular}
\end{center}

\noindent
These simplify to:
\begin{align*}
  g_1(\bx) &= \exp(e - \ln x_1) = e \cdot x_1^{-1} \cdot \exp(0) \quad
    \text{(involves $1/y$)}, \\
  g_2(\bx) &= \exp(e - \ln x_0) \quad
    \text{(involves $1/x$)}, \\
  g_4(\bx) &= \exp(e - \ln x_1) - \ln x_2 \quad
    \text{(mixes exponential and logarithmic)}.
\end{align*}
The model predominantly discovers observables built from $\eml(1, x_i) = e - \ln x_i$
as an inner expression, then applies a second EML layer.  This structure is natural:
depth-2 trees can represent $e^x$, $\ln x$, and their compositions, but not
yet multiplication or addition (which require depth $\ge 5$).  The abundance of
logarithmic and exponential-reciprocal features suggests the model has found the
most informative nonlinear transformations available at depth~2.

\paragraph{Interpretability advantage.}
Unlike the 43{,}299-parameter Deep Koopman autoencoder --- whose observables are
opaque 3-layer neural networks --- every EML observable is a human-readable formula.
The resulting Koopman eigenfunctions $\varphi_j(\bx) = \bv_j^\top \bg(\bx)$ are
therefore explicit linear combinations of elementary functions, amenable to analytic
simplification and physical interpretation.

\section{Discussion}
\label{sec:discussion}

\subsection{Eigendecomposition and Interpretability}

After training and snapping, the explicit Koopman matrix~$\bK$ admits
eigendecomposition:
\begin{equation}\label{eq:eigendecomp}
  \bK = \bV \,\Lambda\, \bV^{-1}, \qquad
  \Lambda = \operatorname{diag}(\lambda_1, \ldots, \lambda_N).
\end{equation}
This yields:

\begin{itemize}
  \item \textbf{Koopman eigenvalues}~$\lambda_j$: encode frequencies
    ($\omega_j = \operatorname{Im}(\ln \lambda_j)$) and growth/decay rates
    ($\sigma_j = \operatorname{Re}(\ln \lambda_j)$).
  \item \textbf{Koopman eigenfunctions:}
    $\varphi_j(\bx) = \bv_j^\top\, \bg_\theta(\bx)$, i.e., linear combinations
    of the snapped EML observables.  Since each $g_i$ is a closed-form elementary
    expression, $\varphi_j$ is also elementary --- fully interpretable.
  \item \textbf{Koopman modes:} $\bm_j = \bC\,\bv_j$, linking each eigenfunction
    to its contribution in state space.
\end{itemize}

\noindent
This is a fundamental advantage over deep-learning Koopman approaches
\citep[e.g., LRAN,][]{otto2019,lusch2018}: instead of opaque neural network
observables, we get symbolic formulas that can be analyzed, simplified, and
compared to physics.


\subsection{Connection to Information Geometry}

The Koopman eigenfunctions $\{\varphi_j\}$ define a natural coordinate system on
the state manifold~$\cM$.  The lifting map
$\bg_\theta : \cM \to \R^N$ induces a pullback metric on~$\cM$:
\begin{equation}\label{eq:pullback}
  (G_\theta)_{ab} \;=\; \sum_{j=1}^{N}
  \frac{\partial g_j}{\partial x^a}\,\frac{\partial g_j}{\partial x^b}.
\end{equation}
When $\bg_\theta$ consists of snapped EML trees, each $g_j$ is a known elementary
function, so $G_\theta$ is computable in closed form.  This connects directly to the
Fisher information metric if the observables are interpreted as sufficient statistics
of a parametric family.

For multi-agent systems, this enables:
\begin{itemize}
  \item \textbf{Behavioral basis decomposition:} The Koopman eigenfunctions separate
    dynamics into independent modes, each with characteristic timescale $1/\sigma_j$
    and frequency~$\omega_j$.
  \item \textbf{Natural gradient on policy manifolds:} The pullback metric~$G_\theta$
    provides the Riemannian structure for natural gradient descent, with an explicit
    elementary form.
  \item \textbf{Geodesic policy interpolation:} Geodesics on
    $(\cM, G_\theta)$ define shortest paths between behavioral strategies.
\end{itemize}


\subsection{Limitations and Future Work}

Several open questions remain:

\begin{enumerate}
  \item \textbf{Complex mode:} Oscillatory dynamics require complex-valued Taylor
    primitives throughout the tree, with careful handling of branch cuts.  The
    implementation (Section~\ref{sec:taylor}) supports this but has not yet been
    validated at scale.
  \item \textbf{Depth scaling:} Convergence from random initialization degrades
    beyond depth~4.  Whether curriculum depth growth or EDMD bootstrapping
    fully resolves this for deep trees ($d \ge 6$) is an open question.
  \item \textbf{Spectral complexity:} Is there a quantifiable relationship between
    EML tree depth and the spectral complexity of the Koopman operator for a given
    system class?
  \item \textbf{EML-KAN hybrid:} Using EML nodes as activation functions in
    Kolmogorov--Arnold networks \citep{liu2025kan} could combine KAN's universal
    approximation with EML's symbolic snap property.
  \item \textbf{Hardware acceleration:} The Horner-form EML primitive maps naturally
    to FPGA or analog circuits, potentially enabling dedicated EML coprocessors that
    evaluate deep trees in $O(d)$ clock cycles.
\end{enumerate}

\section{Conclusion}
\label{sec:conclusion}

We have presented a framework for learning Koopman operator representations using
EML observable dictionaries --- parameterized binary trees of the universal
$\eml(x,y) = \exp(x) - \ln(y)$ operator.  The completeness of EML over elementary
functions provides a principled, gradient-trainable search space that is guaranteed
to contain Koopman eigenfunctions whenever they are elementary.  GPU-optimized
Horner-form Taylor primitives eliminate transcendental function calls, and
Gumbel-softmax annealing with three-phase training produces exact symbolic
observables upon convergence.

On the CTF Lorenz benchmark, EML-Koopman achieves the best short-term forecasting
score (E1 $= 7.68$) among all implemented baselines and surpasses the two other
interpretable CTF methods (SINDy at $-3.00$, PyKoopman at $-20.11$), while using
only 720 parameters --- 60$\times$ fewer than the Deep Koopman autoencoder.  Every
discovered observable is a closed-form symbolic expression, yielding interpretable
Koopman eigenfunctions that black-box methods cannot produce.

The framework opens several directions for future work: scaling to deeper trees
(depth 3--5) via curriculum training to close the gap with high-capacity models,
complex-valued extensions for oscillatory systems, hybrid EML-KAN architectures,
and hardware-accelerated EML evaluation on FPGAs.
